/schedule
Công dụng
Thực hiện Function 1 lần khi Chunk được Load qua tầm của Player hoặc qua tickingarea
Đối tượng có thể dùng Command
Player có OP
Chủ Map
Addon
/execute
/function
/gametest
Command Block
Cú pháp
/schedule on_area_loaded add <from: x y z> <to: x y z> <function: filepath>
on_area_loaded là Chunk được Load
add là tạo
<from: x y z> là tọa độ bắt đầu
<to: x y z> là tọa độ kết thúc
<function: filepath> là tên tệp tin có đuôi .mcfunction
/schedule on_area_loaded add 0 0 0 10 0 10 test — Function tên test sẽ chạy khi tọa độ 0 0 0 đến 10 0 10 thuộc Chunk (0,0;15,15) trong tầm của Player
/schedule on_area_loaded add circle <center: x y z> <radius: int> <function: filepath>
on_area_loaded là Chunk được Load
add là tạo
circle là hình tròn nhưng trong Game nó là hình vuông
<center: x y z> là tọa độ trung tâm
<radius: int> là bán kính
<function: filepath> là tên tệp tin có đuôi .mcfunction
/schedule on_area_loaded add circle 0 0 0 2 test — Function tên test sẽ chạy khi có trung tâm tọa độ 0 0 0 thuộc Chunk (0,0;15,15) bán kính 2 Chunk là 5 x 5 Chunk trong tầm của Player
/schedule on_area_loaded add tickingarea <name: string> <function: filepath>
on_area_loaded là Chunk được Load
add là tạo
tickingarea là Load Chunk chỉ tính x z khi ngoài tầm nhìn của Player
<name: string> là id của tickingarea
<function: filepath> là tên tệp tin có đuôi .mcfunction
/schedule on_area_loaded add tickingarea a test — Function tên test sẽ chạy khi tickingarea tên a tồn tại
Lưu ý
Command có thể được lưu cộng dồn nếu chưa thực hiện được
Ví dụ 1: nếu dùng 3 lần schedule khi Chunk chưa được Load thì sẽ lưu lại schedule đó cho đến khi nào Chunk được Load sẽ chạy 3 lần schedule đó
Ví dụ 2: nếu dùng 3 lần schedule khi tickingarea chưa tồn tại thì sẽ lưu lại schedule đó cho đến khi nào tickingarea tồn tại sẽ chạy 3 lần schedule đó
